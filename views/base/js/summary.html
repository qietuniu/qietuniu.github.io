# 一. JavaScript 中，=== 和==的区别是什么？描述一下判断中间的比较过程？都有哪些隐式转换规则？

# 1. 解答

# 1-1. === 和==的区别

使用==比较时：先检查两边的操作数的数据类型，如果相同直接比较，如果不同按照隐式转换规则进行转换，转换成相同类型再进行比较。 使用===比较时：如果类型不同直接不等，不做类型转换

# 1-2. ==的隐式转换规则

值类型(undefined, null, number, bool, string)间比较，数值优先级最高。 值类型与引用类型(Object, Array, RegExp, Date, Function)间比较，引用类型转成值类型进行比较 。

# 二.  0. 1 + . 2 !== . 3 ? 为什么？

# 1. 解答

计算机精度问题，0. 1转换成2进制后变成一个无限循环小数，根据js使用国际标准 IEEE 754，转成科学计数法后只能保留52个尾数。

0.1 + 0.2 // 0.30000000000000004
1

# 2. 引申

# 2-1. 精度问题的解决方法

  1. toFixed()
  2. Number.EPSILON
let yourSet = 52 - 10; //精度设置为小数点后10位小数
let yourTolerance = Number.EPSILON * Math.pow(2, yourSet);

2

# 3. 思考题

# 3-1. 为什么0. 1 + 0. 1 === 0. 2

答：两个有舍入误差的值在求和时，相互抵消了，但这种“负负得正，相互抵消”不一定是可靠的，当这两个数字是用不同长度数位来表示的浮点数时，舍入误差可能不会相互抵消。

  1. 1 + 0. 3 ，结果其实并不是0. 4，但0. 4是最接近真实结果的数，比其它任何浮点数都更接近。许多语言也就直接显示结果为0. 4了，而不展示一个浮点数的真实结果了 二进制能精确地表示位数有限且分母是2的倍数的小数，比如0. 5在计算机内部就没有舍入误差

# 三. [0, 1, 2]. map(parseInt)的输出是什么？为什么？

# 1. 解答

输出：[1, NaN, NaN] parseInt需要两个参数，但map传了三个参数，等价于[parseInt("1", 0), parseInt("2", 1), parseInt("3", 2)]  parseInt("1", 0)：按照十进制输出1=>1 parseInt("2", 1)：按照1进制输出2, 不存在1进制=>NaN parseInt("3", 2)：按照2进制输出3, 二进制只有01=>NaN

# 2. 引申

# 2-1. map

# 定义

按照原始数组元素顺序依次处理元素，然后返回一个新数组。

  • 不检测空数组
  • 不改变原数组
# 语法

array. map(function(item, index, array), this)

item： 必须，当前处理的数组的值 index：可选，当前处理的值得索引 array：可选，当前处理的值所属于的数组对象 this：可选。对象作为该执行回调时使用，传递给函数，用作 "this" 的值。如果省略了 thisValue ，"this" 的值为 "undefined"。

# 代码
Array.prototype.myMap = function(fn, thisValue) {
    let arr = thisValue || this
    let result = []
    if (typeof fn !== 'function') {
        throw new TypeError(fn + ' is not a function');
    }
    for (let i = 0; i < arr.length; i++) {
        let r = fn.call(arr, arr[i], i, arr)
        result.push(r)
    }
    return result
}

# 2-2. parseInt

# 定义

解析一个字符串，并返回一个整数。当参数 radix 的值为 0，或没有设置该参数时，parseInt() 会根据 string 来判断数字的基数。

默认基数 规则
以 "0x" 开头 把 string 的其余部分解析为十六进制的整数
以 0 开头 把其后的字符解析为八进制或十六进制的数字
以 1 ~ 9 的数字开头 解析为十进制的整数
字符串有多个数字 只返回第一个，例如parseInt("10. 33")   //返回10；
字符串有空格 开头和结尾的空格是允许，例如parseInt(" 69 10 ")   //返回69
第一个字符不能被转换为数字 返回 NaN。例如parseInt("f")  //返回NaN  parseInt("f"，16)  //返回15
#
# 语法

parseInt(string, radix)

  • string: 必须，要被解析的字符串
  • radix：可选。表示要解析的数字基数，介于2-36之间。
# 示例
console.log(parseInt(1, 0)); //将十进制数1转化为十进制数，仍为1
console.log(parseInt(2, 1)); //将一进制数2（不存在）转化为十进制数，NaN
console.log(parseInt(3, 2)); //将二进制数3（不存在）转化为十进制数，NaN
console.log(parseInt(4, 3)); //将三进制数4（不存在）转化为十进制数，NaN
console.log(parseInt(10, 4)); //将四进制数10转化为十进制数为4

# 七. 手写柯里化函数

# 1. 解答

function curry(fn, currArgs) {
    return function() {
        let args = [].slice.call(arguments);
        // 首次调用时，若未提供最后一个参数currArgs，则不用进行args的拼接
        if (currArgs !== undefined) {
            args = args.concat(currArgs);
        }
        // 递归调用
        if (args.length < fn.length) {
            return curry(fn, args);
        }
        // 递归出口
        return fn.apply(null, args);
    }
}

# 八. 手写Reduce函数

# 1. 解答

# 定义

reduce() 方法接收一个函数作为累加器，数组中的每个值（从左到右）开始缩减，最终计算为一个值。 reduce() 可以作为一个高阶函数，用于函数的 compose。

注意: reduce() 对于空数组是不会执行回调函数的。

# 语法

array. reduce(function(total, currentValue, currentIndex, arr), initialValue)

# 代码
Array.prototype.myReduce = function(fn, initialValue) {
    let arr = this
    if (typeof fn !== 'function') {
        throw new TypeError(fn + ' is not a function');
    }
    if (!arr.length) {
        throw new TypeError('数组不能为空');
    }
    let result = initialValue || 0
    for (let i = 0; i < arr.length; i++) {
        result = fn.call(arr, result, arr[i], i, arr)
    }
    return result
}

# 2-1. Every

定义 every() 方法用于检测数组所有元素是否都符合指定条件（通过函数提供）。 every() 方法使用指定函数检测数组中的所有元素： 如果数组中检测到有一个元素不满足，则整个表达式返回 false ，且剩余的元素不会再进行检测。 如果所有元素都满足条件，则返回 true。

注意: every() 对空数组检测 返回 true。every() 不会改变原始数组。

语法 array. every(function(currentValue, index, arr), thisValue)

代码

Array.prototype.myEvery = function(fn, thisValue) {
    let arr = thisValue || this // thisValue 有值 就以thisValue 为准
    if (typeof fn !== 'function') {
        throw new TypeError(fn + ' is not a function');
    }
    // 空数组 返回 true
    if (!arr.length) {
        return true
    }
    for (let i = 0; i < arr.length; i++) {
        if (!fn.call(this, arr[i], i, arr)) {
            return false
        }
    }
    return true
}

# 1-2. 高阶函数

高阶函数：可以传入另外一个函数作为参数的函数。 本身是函数，参数是函数，返回是函数。常用高阶函数有map，reduce，filter，sort等

# 1-3. 高阶组件

高阶组件类似于高阶函数，接收 React 组件作为输入，输出一个新的 React 组件。高阶组件让代码更具有复用性、逻辑性与抽象特征。可以对 render 方法作劫持，也可以控制 props 与 state，实现方法如下：

  1. 属性代理(props proxy)。属性组件通过被包裹的 React 组件来操作 props。
  2. 反向代理(inheritance inversion)。高阶组件继承于被包裹的 React 组件。
